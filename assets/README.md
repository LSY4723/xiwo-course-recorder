# Assets 目录说明

本目录用于存放应用所需的各种资源文件，如图标、图片、音频等。

## 目录结构

```
assets/
├── icons/              # 应用图标
│   ├── app-icon.png    # 主应用图标
│   ├── tray-icon.png   # 托盘图标
│   └── splash.png      # 启动画面
├── images/             # 界面图片
│   ├── logo.svg        # 品牌Logo
│   ├── background.jpg  # 背景图片
│   └── illustrations/  # 插图资源
├── audio/              # 音频文件
│   ├── start-recording.mp3  # 开始录制提示音
│   ├── stop-recording.mp3   # 停止录制提示音
│   └── error.mp3            # 错误提示音
└── fonts/              # 自定义字体
    └── custom-font.ttf
```

## 图标规范

### 应用图标
- **格式**: PNG
- **大小**: 至少 512x512 像素
- **建议**: 提供多种尺寸的图标（16x16, 32x32, 64x64, 128x128, 256x256, 512x512）
- **命名**: app-icon-{size}.png

### 托盘图标
- **格式**: PNG
- **大小**: 16x16, 32x32, 64x64 像素
- **要求**: 支持透明度，背景透明
- **命名**: tray-icon-{size}.png

### 启动画面
- **格式**: PNG
- **大小**: 根据目标平台设计
  - Windows: 1280x720 像素
  - macOS: 1440x900 像素
  - Linux: 1280x720 像素
- **要求**: 简洁明了，突出品牌

## 图片资源

### Logo
- **格式**: SVG 矢量图 + PNG 位图
- **大小**: 至少 256x256 像素
- **要求**: 提供透明背景版本

### 背景图片
- **格式**: JPG 或 PNG
- **大小**: 根据设计需要
- **优化**: 确保文件大小在合理范围内（建议 < 500KB）

## 音频资源

### 提示音
- **格式**: MP3 或 WAV
- **时长**: 建议 1-3 秒
- **要求**: 清晰、悦耳，不要过长
- **音量**: 适中，不要过大

## 字体资源

### 自定义字体
- **格式**: TTF 或 OTF
- **许可证**: 确保有合法的使用权限
- **大小**: 单个字体文件建议 < 1MB

## 资源优化建议

### 图片优化
1. 使用压缩工具优化图片（如 TinyPNG, Squoosh）
2. 对于不同分辨率屏幕提供不同尺寸的图片
3. 优先使用 SVG 矢量格式的图标和插图

### 音频优化
1. 使用合适的比特率（建议 128-256 kbps）
2. 裁剪不必要的静音部分
3. 优先使用 MP3 格式以减小文件大小

## 版权注意事项

1. **自有资源**: 确保所有自有资源的版权清晰
2. **第三方资源**: 所有使用的第三方资源必须有合法的使用权限
3. **开源资源**: 遵循相应的开源许可证
4. **商业授权**: 商业用途的资源需要获得明确的授权

## 添加新资源

1. 将资源文件复制到对应的子目录
2. 在代码中使用相对路径引用
3. 确保文件名使用小写字母和连字符（避免空格和特殊字符）
4. 更新文档记录新增的资源

## 工具推荐

### 图片处理
- **矢量编辑**: Adobe Illustrator, Inkscape
- **位图编辑**: Adobe Photoshop, GIMP
- **图标制作**: Figma, Sketch, IconJar
- **压缩工具**: TinyPNG, Squoosh, ImageOptim

### 音频处理
- **录制工具**: Audacity, Adobe Audition
- **格式转换**: FFmpeg, Online Audio Converter
- **编辑工具**: Audacity, WavePad

### 字体工具
- **字体管理**: FontBase, Suitcase Fusion
- **在线字体**: Google Fonts, Adobe Fonts

## 部署注意事项

1. **Electron 打包**: 确保所有资源文件都在 electron-builder 的打包范围内
2. **文件路径**: 使用 path.join() 处理跨平台路径
3. **缓存策略**: 对于不经常变化的资源可以考虑缓存
4. **加载速度**: 优化资源加载速度，避免影响应用启动时间
